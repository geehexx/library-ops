# Library Ops Core Spec

## Status

Supporting Spec Kit artifact. The canonical product contract is `.taskmaster/docs/prd.md`; this file summarizes the implementation slice for Spec Kit consistency and agent execution.

## User-visible goal

For the current Phase 1 branch, build the first evaluator-visible Django slice
where an evaluator can log in as Admin, Librarian, or Member; see role-aware
navigation; browse a small catalog foundation surface backed by real domain
models; and confirm that protected creation paths are server-side restricted.

## Primary participants

- **Admin**: manages users, roles, catalog records, and system evidence.
- **Librarian**: manages catalog/inventory and performs circulation workflows.
- **Member**: searches catalog, views availability, and sees their own loans.
- **Evaluator**: verifies assignment coverage, local setup, deployed demo, and quality evidence.

## Core scenarios

1. Admin/librarian signs in and reaches a role-aware home/dashboard surface.
2. Admin/librarian creates a foundation catalog record that persists a work,
   contributor, edition, and copy relation.
3. Member signs in and can browse the catalog foundation surface but cannot
   mutate it.
4. Anonymous and Member direct POST attempts to the protected create flow fail.
5. Seed commands create deterministic roles and disposable demo users.
6. The database enforces one active loan per copy even before the later
   circulation workflow branch ships.

## Non-negotiable requirements

- Work, edition, copy, and loan are separate concepts.
- A copy cannot have more than one active loan.
- Availability is derived from database state, not a hand-edited display flag.
- Authorization is enforced server-side.
- Role/demo-user seed data is generated by idempotent management commands, not
  committed fixtures.
- The selected Codex/Task Master/Spec Kit/RTK/code-intelligence toolchain is required for implementation environments.

## Deferred from this branch

- borrow/return UI and service flows
- exact/FTS/vector search behavior
- public-domain import pipeline
- AI metadata suggestions
- deployment and release evidence
