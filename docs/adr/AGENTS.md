# ADR agent rules

- Keep ADRs for decisions that materially affect structure, quality
  attributes, external dependencies, operational posture, or tool policy.
- Use the template format from `template.md` when creating ADRs
- Do not use ADRs for runbook steps, transient notes, or routine doc edits.
- Preserve provenance in each ADR: context, decision, alternatives,
  consequences, and validation.
- Update `index.md` when ADR files are added, removed, renamed, or reclassified.
- Prefer `../../SETUP.md`, `../ARCHITECTURE.md`, and `../README.md` for
  durable operational and explanatory content.
