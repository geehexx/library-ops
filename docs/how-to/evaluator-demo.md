# Evaluator demo path

An evaluator should be able to answer these questions quickly:

1. What is Library Ops?
2. What is implemented versus planned?
3. How do I run or verify it?
4. What is blocked by provider, auth, or environment constraints?
5. How do I trust the current repo evidence?

## Suggested demo flow

1. Read `README.md`.
2. Open `docs/design/wireframes.md` and `docs/design/evaluator-design-handoff.md`.
3. Review `.taskmaster/docs/prd.md`.
4. Review current validation commands in `docs/runbook.md` and `docs/process/quality-gates.md`.
5. Review the current Task Master notes and any intentionally shared local validation outputs for the checks the evaluator cares about.

## Required visible blockers

Do not bury blockers in long prose. If Task Master or provider-backed Promptfoo could not complete, surface that immediately in the current summary of repo state and the related Task Master notes.
