# Community Preset Audit Skill

## Required evaluation

For each preset:
- relevance to Library Ops
- templates/commands/scripts count
- whether it requires another extension
- source review status
- security/tooling risk
- install/adapt/reject decision
- ADR or constitution impact

## Default policy

- Do not install community presets without review.
- Prefer adapting governance concepts into PRD/AGENTS/skills unless the preset materially improves Spec Kit generation.
- Reject creative-writing, Azure-specific, Jira-only, migration-only, and multi-repo presets for this project unless scope changes.
